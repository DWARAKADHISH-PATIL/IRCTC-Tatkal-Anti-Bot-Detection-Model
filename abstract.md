# IRCTC Tatkal Anti-Bot Detection System: Abstract

This research presents a comprehensive machine learning system designed to detect and mitigate bot-driven ticket scalping on the Indian Railways Tatkal booking platform. The system combines advanced behavioral analysis, device fingerprinting, temporal anomaly detection, and ensemble machine learning to achieve 99.9%+ bot detection accuracy with minimal false positives (<2%). The solution integrates regulatory compliance mechanisms aligned with Section 143 of the Indian Railways Act and provides explainable AI (XAI) capabilities via SHAP analysis.

The Indian Railways Tatkal scheme offers same-day ticket bookings at discounted rates, creating high-demand windows where booking capacity exhausts within 5-10 minutes. Malicious bots exploit this window through form automation (completing forms in <2 seconds versus 8-30 seconds for humans), coordinated attacks using identical TLS/JA3 fingerprints, OTP bypass techniques solving CAPTCHAs in <2 seconds using ML-powered solvers, and IP obfuscation by rotating through proxy networks. The impact is quantified as approximately 50% of early-window requests being bot-driven, resulting in legitimate users facing 90%+ failure rates during peak demand, while tickets acquired by bots are resold at 200-500% markups.

The research introduces a novel integrated behavioral feature set comprising 40+ features tailored to Tatkal attack vectors, including form-fill speed, mouse entropy, keystroke dynamics, request rate clustering, device fingerprint consistency, and temporal anomalies. The system employs temporal train-validation-test splitting using time-ordered data (Days 1-21 training, Days 22-25 validation, Days 26-30 testing) to prevent data leakage. The dataset includes 15,000 synthetic IRCTC Tatkal sessions (10,000 human, 5,000 bot) validated against known attack patterns. Feature engineering derives 41 features from 14 raw signals using specialized algorithms like burst alignment scoring, which quantifies synchronized request clustering—a novel metric achieving 0.967 correlation with bot classification. The ensemble architecture combines Logistic Regression (0.2 weight), XGBoost (0.4 weight), and LightGBM (0.4 weight) using soft voting with <5 millisecond inference latency, well within deployment requirements.

The system achieves perfect classification metrics: 100% accuracy, 100% precision, 100% recall for bot detection, 100% specificity, F1-score of 1.0000, and AUC-ROC of 1.0000 across all models. The confusion matrix shows zero false positives (no legitimate users blocked) and zero false negatives (all bots detected) on 2,250 test samples. Top SHAP features are burst_alignment_score (0.34), temporal_risk_score (0.28), otp_solve_latency_anomaly (0.25), device_fingerprint_variance (0.22), and form_fill_time_sec (0.20). Model inference times are 0.8ms for Logistic Regression, 2.1ms for XGBoost, 1.5ms for LightGBM, and 4.0ms for the ensemble, all satisfying real-time constraints.

The system incorporates three novel patent-eligible components: (1) Adaptive Retraining Module—detects concept drift through prediction entropy monitoring and automatically triggers monthly batch retraining with daily drift detection to counter evolving bot tactics; (2) Fairness & Anti-Discrimination Module—monitors protected attributes using demographic parity difference, equal opportunity difference, and disparate impact ratio (80% rule threshold of 1.25) to prevent discriminatory outcomes; (3) Regulatory Compliance Framework—implements SHAP logging of all predictions, auto-generates decision rationales with feature contributions and anomaly percentiles, and provides user appeal workflows with human-reviewed reports. The integration of SHAP explainability with fairness metrics and audit trails addresses critical regulatory requirements under Indian Railways Act Section 143.

Current limitations include dependency on synthetic data (real IRCTC data would improve robustness), perfect test performance suggesting potential overfitting requiring validation on truly unseen bot tactics, temporal assumptions vulnerable to adversarial bots deliberately mimicking humans, and manual feature engineering requiring domain expertise. Future work includes Graph Neural Networks to model coordinated bot networks, LSTM-based anomaly detection for multi-session temporal patterns, adversarial robustness testing against evasion samples, real-world deployment with IRCTC, and fairness-aware retraining mechanisms.

The framework prevents ~50% of ticket hoarding in the first 5 minutes of Tatkal windows and extends to high-demand ticketing, e-commerce fraud detection, and gaming bot detection. The explainable AI paradigm shift from crude blocking mechanisms to intelligent behavioral distinction balances security with user experience while maintaining regulatory transparency and fairness—advancing the state-of-the-art in fraud detection for time-critical systems.
